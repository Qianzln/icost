# 08. 视觉设计规范

## 1. 设计风格

整体风格参考用户截图中的现代记账 App 设计语言：

- 浅灰背景
- 白色大圆角卡片
- 大字号金额
- 图标分类清晰
- 底部悬浮导航
- 记账页底部弹出
- 设置页分组列表

产品要给人一种“干净、轻松、可信赖、有生活感”的感觉，避免过度金融化和复杂化。

## 2. 色彩规范

| 用途 | 色值 | 说明 |
|---|---|---|
| 主色 | #3A9CFF | 选中态、主按钮、重点入口 |
| 背景色 | #F5F5F5 | 页面背景 |
| 卡片色 | #FFFFFF | 卡片、列表容器 |
| 支出色 | #F45D55 | 支出金额、删除、超支 |
| 收入色 | #35C77B | 收入金额、正向增长 |
| 警告色 | #FFC928 | 预算预警、会员标识 |
| 主文字 | #111111 | 标题、金额、主要文本 |
| 次文字 | #666666 | 描述、辅助信息 |
| 弱文字 | #999999 | 占位、低优先级信息 |
| 分割线 | #EEEEEE | 列表分隔 |

## 3. 字体规范

| 场景 | 字号建议 | 字重 |

| 页面大标题 | 28-32px | 700/800 |
| 金额主数字 | 40-52px | 700/800 |

| 列表标题 | 15-16px | 500/600 |
| 正文 | 14-16px | 400/500 |
| 辅助说明 | 12-13px | 400 |
| 底部导航 | 11-12px | 500 |

## 4. 圆角与间距

| 元素 | 建议值 |
|---|---|
| 页面左右边距 | 16px |
| 卡片内边距 | 16-20px |
| 大卡片圆角 | 24px |

## 6. 关键组件规范

### 6.1 月度总览卡片

内容：

- 当前月份
- 总支出或净资产主金额
- 总收入
- 结余
- 预算剩余

视觉：

- 白色卡片
- 大圆角
- 主金额大字号
- 次级数据横向排列

### 6.2 流水列表项

内容：

- 分类图标
- 分类名称
- 备注
- 成员头像
- 账户
- 金额
- 共同/个人/私密标识

交互：

- 点击进入详情
- 左滑删除或更多操作，可选
- 长按复制为模板，可选

### 6.3 快速记账面板

视觉：

- 底部弹出
- 顶部圆角
- 分类图标网格
- 金额区域突出
- 数字键盘大按钮

按钮：

- 完成按钮使用支出色或主色
- 保存再记使用浅色按钮
- 删除按钮使用黑色或深灰小图标

### 6.4 底部导航

视觉：

- 悬浮胶囊样式
- 选中项有浅色背景
- 中间或右侧“+”按钮突出
- 图标 + 文字组合

## 7. 状态设计

### 空状态

示例：

```text
本月暂无账单
点 + 记录第一笔家庭支出吧
```

### 加载状态

- 使用骨架屏
- 卡片区域先展示轮廓
- 避免全屏 loading 时间过长

### 错误状态

示例：

```text
同步失败，请检查网络后重试
```

### 成功状态

示例：

```text
已保存
已生成待结算 ¥50
```

## 8. 动效建议

- 记账页从底部弹出，时长 250ms 左右
- 分类选中时轻微缩放
- 保存成功后轻提示
- 统计数字进入页面时轻微递增
- 待结算标记完成后卡片向上收起

## 9. 可访问性建议

- 金额和主要按钮对比度要足够高
- 颜色不要作为唯一信息表达，需配合文字
- 点击区域不小于 44px
- 字体大小支持系统缩放
