# 07. MVP 版本规划

## 1. MVP 目标

第一版的目标不是做一个功能大而全的记账软件，而是验证核心场景：

> 情侣或家庭是否愿意一起使用一个账本，并通过它解决共同支出和分摊结算问题。

核心验证问题：

- 用户是否愿意邀请另一半或家庭成员？
- 用户是否能快速完成第一笔共同支出？
- 分摊结算是否真的减少沟通成本？
- 成员维度统计是否有吸引力？

## 2. MVP 必做功能 P0

### 2.1 用户与账本

- 手机号或第三方登录
- 创建账本
- 设置账本名称、图标、币种
- 邀请 1 位成员
- 切换账本

### 2.2 快速记账

- 新增支出
- 新增收入
- 新增转账
- 选择分类
- 输入金额
- 选择账户
- 选择付款人
- 选择共同支出/个人支出
- 填写备注和时间

### 2.3 流水管理

- 月度流水列表
- 日期分组
- 流水详情
- 编辑流水
- 删除流水
- 搜索账单
- 基础筛选

### 2.4 分摊结算

- 双人平均分摊
- 一人承担
- 自动生成待结算金额
- 待结算页面
- 标记已结算

### 2.5 统计

- 月度支出
- 月度收入
- 月度结余
- 分类支出排行
- 成员付款统计
- 待结算统计

### 2.6 基础设置

- 个人资料
- 成员管理
- 分类管理
- 账户管理
- 默认账户设置
- 默认分摊规则设置

## 3. MVP 可延后 P1

- 完整资产负债管理
- 预算管理
- 周期账单
- 标签管理
- 图片上传
- 数据导入
- 数据导出
- 共同目标
- 多成员家庭
- 多账本高级管理

## 4. MVP 暂不做 P2

- 智能截图识别
- AI 自动分类
- 发票识别
- 多币种实时汇率
- 高级报销流程
- 模板市场
- 银行卡自动同步
- 复杂会员体系

## 5. 推荐版本节奏

### V0.1：原型验证版

目标：验证产品方向和交互路径。

交付：

- 低保真原型
- 关键页面流程
- 5 到 10 对情侣/夫妻用户访谈

### V0.2：内测可用版

目标：支持真实用户完成共同记账。

功能：

- 登录
- 创建账本
- 邀请成员
- 快速记账
- 双人分摊
- 待结算
- 基础统计

### V1.0：正式上线版

目标：形成完整记账闭环。

功能：

- 优化 UI
- 增加预算
- 完善资产
- 完善统计
- 增加导入导出基础能力
- 支持会员基础权益

### V1.1：增强版

目标：提高留存和付费转化。

功能：

- 周期账单
- 共同目标
- 高级统计
- 数据备份
- 多账本
- 私密账单增强

## 6. MVP 页面清单

第一版建议先完成以下页面：

```text
1. 登录页
2. 创建账本页
3. 邀请成员页
4. 账本首页
5. 快速记账页
6. 流水详情页
7. 待结算页
8. 统计页
9. 设置页
```

## 7. MVP 成功指标

| 指标 | 目标 |
|---|---|
| 新用户首日记账率 | > 50% |
| 邀请成员转化率 | > 30% |
| 7 日留存率 | > 25% |
| 双人账本周活跃率 | > 35% |
| 人均每周记账次数 | > 5 次 |
| 分摊功能使用率 | > 20% |
| 待结算标记完成率 | > 30% |

## 8. 第一版开发优先级建议

```text
第 1 周：登录、账本、成员、基础数据结构
第 2 周：快速记账、流水列表、详情编辑
第 3 周：分摊结算、待结算页
第 4 周：统计页、设置页、基础测试
第 5 周：UI 优化、埋点、内测准备
```
